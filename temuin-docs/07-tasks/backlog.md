# Backlog Reference - Temuin

Dokumen ini adalah referensi lintas sprint. Status hidup tetap dikelola di file sprint, bukan di backlog ini.

## Ringkasan Per Sprint

| Sprint | Fokus Utama | Output Minimum |
|-------|-------------|----------------|
| 01 | Fresh start scaffold | Backend dan frontend foundation hidup |
| 02 | Auth dan item flow | Login, list item, detail, create item |
| 03 | Claim dan fitur inti | Claim, search/filter, notifications |
| 04 | Docker dan UTS | Flow inti jalan di Docker |
| 05 | CI dan tests | Backend tests dan CI dasar |
| 06 | Deploy dan microservices | App live dan split service awal |
| 07 | Gateway dan monitoring | Gateway, health, audit, logging |
| 08 | Security dan final | Final regression, hardening, demo akhir |

## Backlog Inti Per Area

### Backend
- Auth dan JWT internal
- Item CRUD dan business rules
- Claim flow
- Master data
- Notifications
- Audit log

### Frontend
- Scaffold React + Tailwind + shadcn/ui
- Auth pages dan protected routes
- Item pages
- Claim pages
- Admin pages
- Final polish dan accessibility

### DevOps
- Env templates
- Dockerfiles dan compose
- CI
- Deploy
- Gateway dan health checks
- Security hardening

### QA & Docs
- Blackbox checks per sprint
- Screenshot bukti
- Regression checks
- Update dokumen aktif bila flow berubah
